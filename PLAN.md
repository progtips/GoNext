## План реализации GoNext

### 1) Подготовка проекта
- Инициализировать Expo SDK 54 + TypeScript
- Подключить Expo Router и React Native Paper
- Настроить структуру `app/`, тему Material 3, базовый `Layout`
- Подключить базовые сервисы Expo: `expo-sqlite`, `expo-location`, `expo-file-system`, `expo-linking`

### 2) Каркас навигации
- Главный экран с названием GoNext и кнопками:
  - Места
  - Поездки
  - Следующее место
  - Настройки
- Экран списка мест
- Экран карточки места
- Экран списка поездок
- Экран деталей поездки
- Экран «Следующее место»

### 3) Модель данных и SQLite
- Спроектировать таблицы:
  - `places`
  - `trips`
  - `trip_places`
  - `place_photos`, `trip_place_photos`
- Реализовать миграции и инициализацию базы
- Создать простой слой доступа к данным (CRUD)

### 4) Режим «Места»
- Добавление места (название, описание, координаты, отметки)
- Список мест с фильтрацией по `visitlater` и `liked`
- Карточка места:
  - описание
  - фото
  - открытие на карте/в навигаторе

### 5) Режим «Поездки»
- Создание поездки (название, даты, описание)
- Добавление мест в маршрут (из базы или новые)
- Управление порядком мест
- Отметка «посещено», заметки, фото по месту
- Просмотр поездки как план/дневник

### 6) Режим «Следующее место»
- Определение активной поездки (`current = true`)
- Поиск первого `visited = false` в `trip_places`
- Экран с данными следующего места
- Кнопки «Открыть на карте» и «Открыть в навигаторе»

### 7) Работа с геолокацией и картами
- Запрос разрешений на геолокацию
- Получение текущих координат
- Открытие внешнего навигатора через `expo-linking`

### 8) Настройки и сервисные действия
- Экспорт/резервное копирование (позже)
- Очистка данных/сброс приложения
- Тогглы UI (тема, системные настройки)

### 9) Полировка MVP
- Проверка офлайн-работы
- Валидация ввода
- Минимальные ошибки и empty states
- Ручное тестирование сценариев пользователя
